# Anti-Patterns

Avoid these behaviors:

- Implementing before reading the relevant local code, official docs, or primary sources.
- Treating one popular GitHub repo as a standard.
- Using stars, citations, or blog popularity as a substitute for evidence.
- Summarizing papers without translating them into engineering constraints.
- Ignoring release dates, breaking changes, deprecations, or archived repos.
- Copying code without checking license and compatibility.
- Reporting "best practice" without showing what source supports it.
- Hiding uncertainty. If evidence is missing, label it.
- Over-researching trivial tasks. If the work is obvious and low-risk, do the work directly.
